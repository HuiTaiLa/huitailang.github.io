🔄 AI流式输出恢复功能测试

⚠️问题描述

当AI正在流式回复过程中，用户离开页面（如返回即时通讯页面），再回到AI聊天页面时，会出现回复错误。用户无法看到完整的AI回复内容。

✅解决方案

实现了页面可见性检测和流式状态恢复机制，确保用户在任何情况下都能看到完整的AI回复内容。

🧪 测试步骤

  1. 打开AI智能助手聊天页面
  2. 发送一个需要较长回复的技术问题
  3. 在AI开始流式回复时，立即点击浏览器后退按钮
  4. 等待几秒钟后，再次进入AI聊天页面
  5. 观察AI回复是否完整显示或继续流式输出

⚡

中断处理

优雅处理网络中断，避免显示错误信息

🔧 技术实现

1. 页面可见性API

  • 使用 document.visibilitychange 事件监听页面状态
  • 页面隐藏时中断流式传输，避免资源浪费
  • 页面重新可见时检查并恢复流式传输

2. 流式状态管理

  • 维护全局流式传输状态：currentStreamingState
  • 记录消息ID、用户消息、机器人ID等关键信息
  • 使用 AbortController 控制流式传输的中断和恢复

3. 消息恢复机制

  • 检查消息历史中的未完成流式消息
  • 重新发起流式API调用，更新现有消息内容
  • 保持消息ID不变，确保UI一致性
测试场景示例：
1. 用户问："请详细介绍5G网络的技术特点和应用场景"
2. AI开始流式回复："5G网络作为第五代移动通信技术..."
3. 用户在回复进行到一半时离开页面
4. 用户重新回到聊天页面
期望结果：
• 如果AI还在生成回复，继续显示流式输出
• 如果AI已完成回复，显示完整的回复内容
• 不显示错误信息，用户体验流畅
