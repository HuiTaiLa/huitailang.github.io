🔧 AI流式输出闪屏问题修复

❌ 问题描述

当用户在AI流式回复过程中离开页面，再回到聊天页面时，会出现以下问题：

  • 闪屏现象：页面快速闪烁，用户体验不佳
  • AbortError错误：控制台显示流式传输被中断的错误
  • 重复中断：使用已中断的AbortController导致立即失败
  • 状态不一致：流式传输状态与实际情况不匹配

错误日志示例：

页面隐藏，中断流式传输
已中断流式传输
流式读取失败: AbortError: BodyStreamBuffer was aborted
AI流式回复失败: AbortError: BodyStreamBuffer was aborted
流式传输被用户中断（页面隐藏）

✅ 解决方案

通过以下技术手段解决了闪屏和错误问题：

  • AbortController管理：正确处理已中断的控制器
  • 状态清理：页面隐藏时清理中断的控制器
  • 智能恢复：页面可见时创建新的控制器
  • 防抖处理：避免快速页面切换的重复操作
  • 错误过滤：不显示用户主动中断的错误信息
🛡️

AbortController管理

页面隐藏时清除已中断的控制器，恢复时创建新的控制器
// 页面隐藏时清理 currentStreamingState.abortController = null; // 恢复时创建新的 currentStreamingState.abortController = new AbortController();
⏱️

防抖处理

延迟页面可见性处理，避免快速切换导致的重复操作
// 延迟200ms处理页面可见 visibilityChangeTimeout = setTimeout(() => { handlePageVisible(); }, 200);
🔍

状态检查

确保只在AI聊天页面进行流式传输恢复
// 检查页面类型 if (currentChatInfo.type !== 'ai_bot') { return; // 跳过恢复 }
🚫

错误过滤

过滤用户主动中断的错误，避免显示误导信息
if (error.name === 'AbortError') { // 不显示错误消息 return; }
