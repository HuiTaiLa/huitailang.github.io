📁 文件协议使用说明

如何在 file:// 协议下使用文档预览功能

🔍 问题说明

⚠️ CORS 安全限制

当使用 file:// 协议直接打开 HTML 文件时，浏览器的安全策略会阻止访问本地文件，导致文档内容无法正常加载。

✅ 解决方案

方案一：使用本地 HTTP 服务器（推荐）

✅ 最佳解决方案

启动一个本地 HTTP 服务器，通过 http://localhost 访问网站。

1 打开终端/命令提示符

在项目根目录下打开终端。

2 启动 HTTP 服务器

根据您的系统选择合适的命令：

Python 3 (推荐):

python3 -m http.server 8000

Python 2:

python -m SimpleHTTPServer 8000

Node.js (需要安装 http-server):

npx http-server -p 8000

PHP:

php -S localhost:8000
3 访问网站

在浏览器中访问：

http://localhost:8000

方案二：手动选择文件（file:// 协议兼容）

💡 备用方案

如果无法启动 HTTP 服务器，系统会自动检测 file:// 协议并提供文件选择功能。

1 系统自动检测

当使用 file:// 协议时，页面会自动显示文件选择界面。

2 选择文档文件

点击"选择文档文件"按钮，选择 uploads 文件夹中的所有文档文件。

3 开始使用

文件选择完成后，即可正常使用文档预览功能。

📋 操作步骤

使用 HTTP 服务器（推荐流程）

  1. 在项目根目录打开终端
  2. 运行命令：python3 -m http.server 8000
  3. 打开浏览器访问：http://localhost:8000
  4. 正常使用所有功能

使用文件协议（备用流程）

  1. 直接双击打开 HTML 文件
  2. 页面会显示文件选择提示
  3. 点击"选择文档文件"按钮
  4. 选择 uploads 文件夹中的所有文档文件
  5. 等待文件缓存完成
  6. 开始使用文档预览功能

🧪 测试页面

您可以使用以下测试页面验证功能：

  • test-real-document-extraction.html - 文档提取功能测试
  • document-viewer.html - 文档查看器测试
  • resource-library.html - 资源库预览测试

❓ 常见问题

Q: 为什么会出现 CORS 错误？

A: 这是浏览器的安全机制，防止恶意网站访问本地文件。使用 HTTP 服务器可以解决这个问题。

Q: 文件选择后为什么还是无法预览？

A: 请确保选择了正确的文档文件，并且文件格式受支持（PDF、DOCX、PPTX）。

Q: 哪种方案更好？

A: 推荐使用 HTTP 服务器方案，功能更完整，性能更好。文件协议方案仅作为备用选择。

Q: 如何知道当前使用的是哪种模式？

A: 查看浏览器地址栏，file:// 开头是文件协议，http:// 开头是 HTTP 协议。

🔧 技术说明

系统会自动检测访问协议：

  • HTTP 协议：直接通过 fetch API 访问 uploads 文件夹
  • File 协议：使用 FileReader API 读取用户选择的文件

🎉 现在您可以开始使用文档预览功能了！

选择合适的方案，按照步骤操作，即可体验真实的文档内容提取和预览功能。
